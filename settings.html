General Investigations

Aperture — Email Forensics

Parse and analyze email evidence from .mbox and .eml files obtained through search warrants or legal process. View full email threads with headers, body content, and attachments in a sandboxed environment. Filter by sender, date range, and keyword search to identify relevant communications.

ARIN Lookup — IP WHOIS

Query the ARIN (American Registry for Internet Numbers) WHOIS database to identify the registered owner, ISP, and network allocation for a given IP address. Returns the organization name, network range, and CIDR block. Automatically follows redirects to other Regional Internet Registries (RIPE, APNIC, etc.) for non-North American IPs. Available on IP address identifiers in the Suspects module.

CDR — Call Detail Records

Import and analyze call detail records from carrier subpoenas. Generates interactive timelines, network relationship graphs, and geolocation maps to visualize communication patterns between targets over time. Supports standard CSV/XLSX tower dump and CDR formats.

Callyo — Undercover Phone & Recording Platform

Callyo (a Motorola Solutions company) provides 10‑21, 25Phones, and Record-A-Call — covert phone numbers, call recording, and undercover communications platforms used by law-enforcement task forces for stings, controlled buys, and informant management.

Email Verifier — SMTP Validation

Verify whether an email address has a valid, deliverable mailbox by performing a real-time SMTP handshake (EHLO → MAIL FROM → RCPT TO) against the domain's mail server. Checks DNS MX records and validates the mailbox exists without sending an actual email. Some providers (Apple/iCloud, Yahoo, ProtonMail) block mailbox-level verification as an anti-spam measure — these are reported as "unverifiable" rather than invalid. Available on email identifiers in the Suspects module.

CargoNet — Theft Alert Ingest

Forward CargoNet Theft Alerts directly into VIPER instead of relying on email. When enabled, VIPER watches a local folder for .eml drops from Outlook, parses the alert (driver info, tractor, cargo, value, investigating agency, incident #), and queues it on the dashboard. One click converts an alert into a fully-populated case (suspects, vehicles, victim business, cargo).

Outlook automation setup (one-time, ~3 minutes)

Pick Method A below — it's a built-in Windows tool that auto-saves every CargoNet alert into this folder the instant it lands in Outlook. No code, no admin, works on gov-tenant mailboxes (uses your local Outlook profile, never the cloud).

Recommended

Method A — Power Automate Desktop flow

Free, pre-installed on Windows 11 (one-click install on Windows 10 from the Microsoft Store). Runs locally — no M365 license, no cloud connector, no admin rights.

  1. Press Win → type Power Automate → open it (install from MS Store if missing).
  2. Click + New flow → name it CargoNet → VIPER → Create.
  3. In the actions panel (left), search outlook. Drag Launch Outlook to the canvas → Save (default settings are fine).
  4. Search again and drag Retrieve email messages from Outlook. Configure:
    • Account: your mailbox
    • Mail folder: Inbox
    • Retrieve: Unread email messages only
    • Mark as read: On
    • From: no-reply@cargonet.com
  5. Drag For each → iterate over %RetrievedEmails%.
  6. Inside the loop, drag Save Outlook email messages. Configure:
    • Save mode: Save as file
    • File format: EML (if EML isn't listed, choose MSG — VIPER also parses .msg)
    • Folder path: (use the inbox folder shown above)
    • Email message: %CurrentItem%
  7. Save the flow. Click … → Properties → Run when I sign in to Windows (sets it to auto-start).
  8. To trigger on a schedule: open the flow → Schedule flow → every 2 minutes → Save.

That's it. CargoNet emails will land here within ~2 minutes of arrival, VIPER ingests them within 30 seconds, and the amber bell on the dashboard lights up.

Method B — Manual drag-and-drop (fallback)

Open the CargoNet email in Outlook → drag it into the inbox folder shown above. Windows Explorer drops a .msg file that VIPER also parses. Useful for one-off alerts or testing.

Method C — CargoNet "Email2" forwarding (advanced)

If your agency runs an inbound mail-to-file service, paste that forwarding address into your CargoNet user profile's Email2 field. Most users should stick with Method A.

VIPER watches this folder every 30 seconds and on filesystem-change events. Once parsed, the source file is moved into archive/ beside the inbox so the same alert is never ingested twice.
